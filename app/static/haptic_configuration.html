Haptic Configuration

Purpose

This page makes the haptic stack explicit. It separates the backend that FeelIT is actively running, the vendor SDKs or bridges that may already be installed on the machine, and the future native runtime needed to drive real hardware inside the same 3D worlds.

Current Interaction

  • Select which backend stack FeelIT should track as the requested runtime.
  • Record SDK roots and bridge executables for OpenHaptics, Force Dimension, or CHAI3D-based paths.
  • Inspect which native build tools are available on the current Windows host for the bridge workflow.
  • Review the native bridge workspace and the commands used to configure or build the bridge scaffold.
  • See whether a bridge executable is missing, failing, scaffold-only, or reporting real device-ready probe data.
  • Inspect the current collision, contact, and material-rendering baseline that the native backend must honor.
  • Keep the built-in visual emulator as the safe default while physical-device integration is still under construction.
